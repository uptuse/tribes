# Firewolf — System Reference

**Last updated:** 1 May 2026  
**Status:** Current as of master HEAD

This document is the authoritative map of every active subsystem. Read it at the start of any new session. It replaces the stale `system-map.md`.

---

## Architecture overview

```
Browser
├── index.html          — Game shell, HUD, menus, settings, multiplayer UI
│   └── imports renderer.js (ES module, lazy)
│
├── renderer.js         — Three.js render loop, scene graph owner
│   ├── renderer_characters.js  — GLB character models + animation
│   ├── renderer_sky.js         — Procedural sky dome + day/night
│   ├── renderer_daynight.js    — Sun/moon cycle, light colours
│   ├── renderer_polish.js      — Weather, shockwaves, impact FX
│   ├── renderer_combat_fx.js   — Muzzle flash, tracers, hit sparks
│   └── client/                 — Editor + networking modules
│
├── tribes.wasm         — C++ game simulation (Emscripten)
│   └── tribes.js               — JS glue generated by Emscripten
│
└── server.js           — Optional Node.js LAN relay (WebSocket + WebRTC signalling)
```

## 1. WASM Physics Engine (`program/code/wasm_main.cpp`)

**What it does:** Runs the entire game simulation — player physics, projectiles, flags, bots, buildings, collision. Exports a flat Float32Array state that the renderer reads every frame via zero-copy views into `HEAPF32`.

**Key exports (callable from JS):**

| Function | Purpose |
|---|---|
| `_tick()` / `mainLoop()` | Advance simulation one frame |
| `_setGameSettings(team, armor, bots, scoreLimit, timeLimit)` | Start/reset match |
| `_setSettings(jsonStr)` | FOV, sensitivity, invertY, jetToggle |
| `_setPhysicsTuning(gravity, jetForce, jetEnergy, friction, speed, kickback)` | Live physics tuning |
| `_pause(int)` | Freeze physics (editor mode) |
| `_setFreelook(int)` | Freeze player aim (camera orbits independently) |
| `_setGamepadInput(fwd, side, lookDX, lookDY, btns)` | Gamepad synthetic input |
| `_teleportPlayer(x, y, z)` | Playtest-from-cursor |
| `_spawnDummy()` | Place 10 stationary enemy targets at red base |
| `_reloadBuildings(jsonPtr, len)` | Hot-reload building collision |
| `_writeHeightmapPatch(cx, cy, radius, strength, mode)` | Sculpt terrain |
| `_applyLoadout(armor, weapon, pack)` | Change player loadout |
| `_getLocalPlayerIdx()` | Index into player state array |
| `_getThirdPerson()` | 1 if third-person camera is active |
| `_setFreelook(int)` | Decouple camera from player aim |

**State arrays read by JS (zero-copy Float32Array views):**

| Array | Stride | Key offsets |
|---|---|---|
| `g_rPlayers[16]` | 32 floats | 0-2=pos, 3-5=rot, 6-8=vel, 9=health, 10=energy, 11=team, 12=armor, 13=alive, 14=jetting, 15=skiing, 16=weaponIdx, 17=numericId, 18=visible, 19=botRole |
| `g_rProjectiles[256]` | 16 floats | 0-2=pos, 3-5=vel, 6=weaponType, 9=alive |
| `g_rParticles[1024]` | 8 floats | 0-2=pos, 6=type, 7=age |
| `g_rFlags[2]` | 8 floats | 0-2=pos, 3=team, 4=state |
| `g_rBuildings[256]` | 16 floats | 0-2=pos, 3-5=halfExtents, 6=type, 7=team, 8=alive |
| `g_heights[257][257]` | — | Mutable terrain heightmap (Sculpt writes here) |

**Known gaps:** No multiplayer authority — each client runs its own simulation. State sync via relay only.

## 2. Renderer (`renderer.js` — 6500 lines)

**What it does:** Owns the Three.js scene graph and render loop. Reads WASM state arrays every frame and drives Three.js objects. Never writes back to WASM except through the exported C++ functions.

**Subsystems inside renderer.js:**

| Subsystem | Function(s) | Notes |
|---|---|---|
| Terrain | `initTerrain()` | 257×257 heightmap → PlaneGeometry, PBR shader |
| Buildings | `initBuildings()` | Loads `canonical.json`, places procedural meshes |
| Interior shapes | `initInteriorShapes()` | Loads `raindance_meshes.bin`, real Tribes .dis geometry |
| Players | `syncPlayers(t)` | Procedural box meshes for all 16 player slots |
| Projectiles | `syncProjectiles()` | Disc=torus, chaingun=tracer box, others=spheres |
| Particles | `syncParticles()` | GPU particle system, 1024 cap |
| Flags | `syncFlags(t)` | 2 flag meshes with team-colour banners |
| Turret barrels | `syncTurretBarrels(t)` | Rotates turret barrel meshes toward targets |
| Camera | `syncCamera()` | 1P/3P, Freelook (Alt), orbital edit camera (OrbitControls) |
| Weapon viewmodel | `initWeaponViewmodel()` + `updateWeaponHand()` | Procedural coloured-frame model, per-weapon frame colour |
| Post-processing | `initPostProcessing()` | UnrealBloom + custom vignette/grade ShaderPass |
| DTS weapons | `initDTSWeaponModels()` | Loads chaingun/disc/grenade DTS geometry (currently disabled) |
| Water | `client/water.js` | Gerstner wave shader, Raindance chasm |

**Quality tiers:** `low / medium / high / ultra`. Auto-downgrade if FPS < 28 sustained. Override with `?quality=low` URL param.

**Coordinate space:** World metres, Y-up. Terrain centred at origin. WASM uses same convention.

## 3. Character System (`renderer_characters.js`)

**What it does:** Loads Mixamo GLB characters, clones instances for the local player, drives animation via `THREE.AnimationMixer`, grounds feet to terrain.

**Architecture (post-R32.277):**

```
_gltf.scene (template, never modified)
└── inner = skeletonClone(_gltf.scene)   ← clone preserves all internal transforms
    └── wrapper = new THREE.Group()       ← receives position/rotation from WASM each frame
        WASM drives wrapper only. Inner transforms (Armature +90°X / Hips -90°X) untouched.
```

**Key exports:**

| Export | Purpose |
|---|---|
| `init(scene)` | Load current character GLB |
| `sync(t, playerView, stride, localIdx, meshes)` | Called every frame |
| `switchCharacter(idxOrId)` | Hot-swap to another roster entry |
| `isLoaded()` | True once GLB is ready |
| `getRig(localIdx)` | Returns `{skeleton, clips}` for animation editor |
| `subscribeRigChange(cb)` | Called when a new rig becomes available |
| `getKind()` | `'skinned'` or `'rigid'` |

**Character roster** (`ROSTER` array in `renderer_characters.js`):
Currently: `crimson_sentinel` only. Others pending Mixamo re-export with "With Skin".

**Asset convention:**
- `<id>_rigged.glb` — Mixamo skeleton + animations + skin weights
- `<id>_50k.glb` — HD PBR textures (transferred to rigged at load time)
- Animation source: crimson sentinel's 14 clips shared to all others via `animSrc`

**Known gaps:** Only `crimson_sentinel` has correct skin weights. All others need re-export. See `docs/CHARACTER_RIG_AUDIT.md`.

## 4. Editor Shell (`client/shell.js` + `client/editor_*.js`)

**What it does:** The Shift+Enter unified editor. 12 mode tiles in a draggable right-side panel. Each mode has a palette that mounts/unmounts into `#fw-palette-host`.

**Modes and their files:**

| Mode | File | Status |
|---|---|---|
| Play | — | Default, physics running |
| Place | `editor_assets.js` | Drop flags/spawns/turrets; delegates to `level_editor.js` |
| Build | `editor_buildings.js` | Wall/floor snap-to-grid placement |
| Sculpt | `editor_terrain.js` | Heightmap brush via `_writeHeightmapPatch` |
| Animate | `editor_animations.js` | Timeline scrubber, clip library, character picker |
| Paint | `editor_materials.js` | Click-to-recolour building meshes |
| Tune | `editor_tuning.js` | Physics sliders → `_setPhysicsTuning` |
| Triggers | `editor_triggers.js` | Region event placement |
| Sound | `editor_audio.js` | Audio event preview/tuning |
| Effects | `editor_vfx.js` | Particle preset spawning |
| Bots | `editor_ai.js` | Bot behaviour placement |
| Bindings | `editor_bindings.js` | Event → VFX/audio reaction wiring |
| Light | `editor_lighting.js` | Sun direction, ambient, fog, terrain brightness |

**Cross-cutting services** (`client/editor_core/`):

| Service | Responsibility |
|---|---|
| `History.js` | Global undo/redo stack (Ctrl+Z / Ctrl+Shift+Z) |
| `Snap.js` | Grid snap (G to cycle, Shift to suppress) |
| `Selection.js` | Box-select, Ctrl+click, group delete/duplicate |
| `CameraBookmarks.js` | Shift+F1–F4 save, F1–F4 recall |
| `MapManager.js` | Save/load maps to localStorage, File dropdown |
| `Layers.js` | Show/hide/lock categories |
| `Playtest.js` | Shift+P teleport to cursor → Play |

**Key keyboard shortcuts:**
- `Shift+Enter` — toggle panel (or return to Play if in edit mode)
- `Esc` — return to Play from any edit mode
- `Shift+P` — playtest from cursor
- `G` — cycle snap mode
- `Ctrl+Z` / `Ctrl+Shift+Z` — undo/redo

**Physics pause:** Entering any edit mode calls `Module._pause(1)`. Exiting calls `Module._pause(0)`.

## 5. Locomotion System (`client/locomotion.js`, `client/foot_ik.js`, `client/camera_grounding.js`)

**What it does:** Makes character movement look grounded — speed-matched stride, camera bob, foot planting.

| Module | Function | Performance note |
|---|---|---|
| `locomotion.js` | `Locomotion.update()` — sets `action.timeScale = speed/naturalSpeed` | Cached per-instance |
| `locomotion.js` | `Locomotion.pelvisBob()` — sine wave on pelvis bone | Bone cached after first find |
| `locomotion.js` | `Locomotion.skiUpdate()` — crouch + lean on ski | Per-frame on local player only |
| `foot_ik.js` | `FootIK.update()` — raycasts (via heightmap) to plant feet | O(1) heightmap lookup |
| `camera_grounding.js` | `CameraGrounding.update()` — footstrike bob + landing kick | Spring-damped, cheap |

**Known gaps:** Foot IK only runs on local player. Ski transitions are procedural, not hand-keyed (looks approximate).

## 6. Networking

### 6a. Local WebSocket relay (`server.js`)
Node.js server. Relays binary game state between browsers on the same LAN. Also serves static game files. Run via `start-server.command` (double-click).

- Port: 3000
- WebSocket path: `/ws`
- Signalling API: `POST /api/signal/offer`, `GET /api/signal/offer/:code`, `POST/GET /api/signal/answer/:code`, `GET /api/ping`

### 6b. WebRTC P2P (`client/browser_host.js`)
Browser-to-browser, no server required for data. Signalling uses the local server (short 6-char codes) or falls back to manual base64 SDP copy-paste.

- Host: Esc → "Host (no server needed)" → share code
- Join: Esc → "Join with code" → paste code

### 6c. Network protocol (`client/network.js`, `client/wire.js`)
Binary wire format. `MSG_SNAPSHOT` and `MSG_INPUT` over WebSocket or WebRTC DataChannel. The DataChannel mimics the WebSocket API so `network.js` is transport-agnostic.

### 6d. Gamepad (`client/gamepad.js`)
Standard browser Gamepad API. Left stick = move, right stick = look, RT = fire, A = jet, B = ski. Calls `Module._setGamepadInput()` each frame.

## 7. Physics Tuning (live, no rebuild)

All sliders in Shift+Enter → **Tune** write via `Module._setPhysicsTuning()`. Changes are instant and survive mode switches. Kickback slider also available.

| Slider | Default | Range |
|---|---|---|
| Gravity | 20 m/s² | 5–40 |
| Jet force | 1.0× | 0.2–3.0 |
| Energy drain | 1.0× | 0.1–3.0 |
| Ground friction | 1.0× | 0.1–2.0 |
| Speed | 1.0× | 0.2–3.0 |
| Kickback | 1.0× | 0.1–5.0 |

## 8. Training Targets (`_spawnDummy()`)

10 stationary heavy-armour enemies placed in a line at the red flag base. Used for weapon testing.

- Spawned automatically when `botCount=0` in `setGameSettings()`
- Respawn individually 1.5s after death
- HP bar displayed above each (T1–T10), values shown as ×100 (132/132 HP = heavy armour)
- Warmup skipped on spawn — all weapons deal damage immediately

---

## 9. Terrain & Water

**Terrain:** 257×257 heightmap from `raindance_heightmap.h`. Mutable at runtime via `_writeHeightmapPatch()` in Sculpt mode. Three.js vertex buffer refreshed after each stroke. C++ collision also uses the mutable `g_heights[]` array.

**Water:** `client/water.js`. Gerstner wave shader at world position (-291.6, 8.5, -296.7) — below the bridge in the Raindance chasm. 400×500m plane, transparent, animated, syncs sun direction to day/night cycle.

---

## 10. Weapons

9 weapons: Blaster (0), Chaingun (1), Spinfusor/Disc (2), Grenade Launcher (3), Plasma (4), Mortar (5), Laser (6), ELF (7), Repair (8).

**Projectile visuals:**
- Disc: spinning blue torus
- Chaingun: yellow tracer box aligned to velocity
- Others: colour-coded spheres (frame colour also shown on viewmodel)

**Weapon viewmodel:** Procedural geometry. Frame colour changes per weapon. Arms use `depthTest:false` so they never clip behind the gun body.

**Damage:** Normalised floats (heavy armour maxDamage=1.32). Display as ×100 for legibility.

## 11. Known Gaps & Technical Debt

| Area | Issue | Priority |
|---|---|---|
| Character models | Only `crimson_sentinel` has correct skin weights. Others need Mixamo re-export with "With Skin" | High |
| Sculpt mode | Heightmap edits not persisted (lost on page refresh) | Medium |
| Multiplayer | No authoritative server — each client runs own sim. State sync is best-effort relay | Medium |
| Animation editor | Standalone at `assets/models/animation_editor.html`. Needs server headers to load on GitHub Pages | Low |
| Collision | Interior building collision uses AABB; complex shapes approximate | Low |
| LOD | LOD switching not implemented — all characters render at 50k poly | Low |
| Asset upload (§9) | No `POST /api/upload` endpoint — file upload is manual | Low |
